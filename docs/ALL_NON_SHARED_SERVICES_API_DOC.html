Auth Service (auth-service)

Base path: /api/v1/auth

POST /api/v1/auth/login/member

Member login (password or OTP). Issues JWT bound to deviceId and includes username claim (email/mobile).

Request (JSON) 
{
  "email": "user@example.com",
  "password": "secret-or-empty-if-otp",
  "deviceId": "device-abc-123",
  "otp": "123456" 
}
Response (success) 
{
  "statusCode":"200",
  "response":"SUCCESS",
  "data": {
    "accessToken":"<jwt-token>",
    "expiresIn":3600,
    "memberCode":"M123"
  }
}

POST /api/v1/auth/logout

Invalidate presented token. Body userId is username (email/mobile). Header Authorization: Bearer <token> required.

Request (JSON) 
{ "userId": "user@example.com" }
Response 
{ "statusCode":"200", "response":"SUCCESS", "data": { "message":"User logged out successfully" } }

Other auth endpoints

Verify OTP, resend OTP, userDetails, biometric endpoints, consent endpoints. All follow same CommonResponse wrapper and use @DecryptBody/@EncryptResponse in production.

Claims Service (claims-service)

Base path: /api/v1/reimbursement

GET /api/v1/reimbursement/history?memberCode={memberCode}

Fetch claims reimbursement history (proxied to reimbursements-service).

Response 
{
  "statusCode":"200",
  "response":"SUCCESS",
  "data": { /* Map of lists keyed by status */ }
}

POST /api/v1/reimbursement/submit (multipart/form-data)

Submit reimbursement with JSON request part and optional files.

Multipart example 
--boundary
Content-Disposition: form-data; name="request"

{"memberCode":"M123","serviceType":"INPATIENT","totalAmount":1234.56}
--boundary
Content-Disposition: form-data; name="serviceInvoice"; filename="inv.pdf"
Content-Type: application/pdf

...binary...
--boundary--
Response 
{ "statusCode":"200","response":"SUCCESS","data": {"controlCode":"C12345","status":"SUBMITTED"} }
