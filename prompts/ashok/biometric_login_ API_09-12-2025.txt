WHAT HAPPENS IN THIS LAYER - STEP BY STEP:

     - Logs warning: `LOGGER.warn("Biometric login failed - Invalid or expired challenge");`

   - Checks: `if (!Boolean.TRUE.equals(user.getBiometricEnabled()))`
   - If not enabled:
     - Deletes challenge: `redisTemplate.delete(challengeKey);` (prevent reuse)
     - Returns error: HTTP 403 FORBIDDEN
     - Message: "Biometric authentication is not enabled"
     - Method stops execution

6. MEMBER User Validation - Status:
   - Checks: `if (!Constants.ACTIVE.equals(user.getStatus().name()))`
   - `getStatus()` returns enum value (e.g., `UserStatus.ACTIVE`)
   - `name()` converts enum to string
   - Compares with `Constants.ACTIVE` string
   - If not active:

     - Deletes challenge

     - Method stops execution

8. HMAC Signature Verification:
   - Computes expected signature: `String expectedSignature = computeHmac(passkeyHash, request.challenge());`

     - Uses `Mac.getInstance("HmacSHA256")` - gets HMAC-SHA256 algorithm
     - Creates `SecretKeySpec` with passkeyHash as key
     - Initializes HMAC with secret key
     - Computes HMAC of challenge bytes
     - Returns Base64-encoded signature

   - If mismatch:
     - Logs warning: `LOGGER.warn("Biometric login failed - Invalid signature");`

     - Returns error: HTTP 401 UNAUTHORIZED with `INVALID_CREDENTIALS` code

2.3 Utility Method - HMAC Computation 

ROLE: You are a backend developer explaining cryptographic utility implementation.

TASK: Explain in detail what happens in the `computeHmac` private utility method.

CONTEXT:
- Method location: `AuthServiceImpl` class
- Method visibility: `private` (only accessible within class)
- Purpose: Compute HMAC-SHA256 signature for challenge verification
- Algorithm: HMAC-SHA256
- Input: `key` (passkeyHash string), `data` (challenge string)
- Output: Base64-encoded HMAC signature string

WHAT HAPPENS IN THIS METHOD - STEP BY STEP:

1. Method Entry:
   - Method signature: `private String computeHmac(String key, String data)`
   - Receives two String parameters:
     - `key`: Passkey hash (SHA256 hash stored in database)
     - `data`: Challenge string (from Redis)
   - Returns: Base64-encoded HMAC signature string

2. Try Block - HMAC Instance Creation:
   - `Mac hmac = Mac.getInstance("HmacSHA256");`
   - `Mac` is Java cryptography class from `javax.crypto` package
   - `getInstance()` is static factory method
   - Parameter "HmacSHA256" specifies HMAC algorithm with SHA-256 hash function
   - Returns `Mac` instance configured for HMAC-SHA256
   - Throws `NoSuchAlgorithmException` if algorithm not available (shouldn't happen)

3. Secret Key Specification:
   - `SecretKeySpec secretKeySpec = new SecretKeySpec(key.getBytes(StandardCharsets.UTF_8), "HmacSHA256");`
   - `SecretKeySpec` is Java cryptography class for secret keys
   - `key.getBytes(StandardCharsets.UTF_8)` converts key string to byte array
   - `StandardCharsets.UTF_8` specifies UTF-8 character encoding
   - Second parameter "HmacSHA256" specifies algorithm name
   - Creates secret key object from passkey hash bytes

4. HMAC Initialization:
   - `hmac.init(secretKeySpec);`
   - `init()` method initializes HMAC with secret key
   - Secret key is set for HMAC computation
   - HMAC is now ready to compute signatures

5. HMAC Computation:
   - `byte[] hmacBytes = hmac.doFinal(data.getBytes(StandardCharsets.UTF_8));`
   - `data.getBytes(StandardCharsets.UTF_8)` converts challenge string to byte array
   - `doFinal()` computes HMAC of data bytes using secret key
   - Returns byte array containing HMAC signature (32 bytes for SHA-256)
   - HMAC algorithm:
     - Combines secret key with data using cryptographic hash function
     - Produces fixed-length output (256 bits = 32 bytes)
     - Cryptographically secure - cannot be reversed or forged

6. Base64 Encoding:
   - `return Base64.getEncoder().encodeToString(hmacBytes);`
   - `Base64.getEncoder()` gets Base64 encoder instance
   - `encodeToString()` converts byte array to Base64-encoded string
   - Base64 encoding converts binary data to ASCII string
   - Result is 44-character string (32 bytes * 4/3 = 42.67, rounded up)
   - This string is returned to caller

7. Exception Handling:
   - `catch (Exception e)` catches any exception:
     - `NoSuchAlgorithmException` - algorithm not available
     - `InvalidKeyException` - invalid secret key
     - `IllegalStateException` - HMAC not initialized
     - Other runtime exceptions
   - Logs error: `LOGGER.error("Error computing HMAC signature", e);`
   - Logs exception stack trace for debugging
   - Throws: `throw new CustomException(HttpStatus.INTERNAL_SERVER_ERROR.value(), "Failed to compute HMAC signature");`
   - `CustomException` is custom exception class
   - HTTP status 500 (Internal Server Error)
   - Exception is caught by `GlobalExceptionHandler` and converted to error response

CLASSES AND METHODS INVOLVED:
- `Mac` - Java cryptography class for MAC (Message Authentication Code)
- `SecretKeySpec` - Java cryptography class for secret keys
- `StandardCharsets` - Java class for character encodings
- `Base64` - Java utility class for Base64 encoding
- `CustomException` - Custom exception class
- `Logger` - SLF4J logger
- `HttpStatus` - Spring HTTP status enum
- `GlobalExceptionHandler` - Exception handler

METHOD SIGNATURE:
```java

```
