Create a private method named handlePeriodType that takes a UtilizationRequest object as input and returns a CommonResponse object.

The method should:

Validate if periodType is null or blank — if true, return:

return CommonResponse.error(new ErrorResponse(400, MemberConstants.PERIODTYPE), 400);


Define LocalDateTime now = LocalDateTime.now();

Use a switch expression (switch (utilizationRequest.getPeriodType().toLowerCase())) to handle the following cases:

"last12months" → set dateFr = now.minusYears(1) and dateTo = now.

"2years" → set dateFr = now.minusYears(2) and dateTo = now.

"3years" → set dateFr = now.minusYears(3) and dateTo = now.

"4years" → set dateFr = now.minusYears(4) and dateTo = now.

"5years" → set dateFr = now.minusYears(5) and dateTo = now.

"custom" →

If either dateFr or dateTo is null, return:

return CommonResponse.error(new ErrorResponse(400, MemberConstants.CUSTOM), 400);


Calculate:

long yearsBetween = ChronoUnit.YEARS.between(utilizationRequest.getDateFr(), utilizationRequest.getDateTo());
if (yearsBetween > 5) {
    return CommonResponse.error(new ErrorResponse(400, MemberConstants.YEARS_BETWEEN), 400);
}


default case → return:

return CommonResponse.error(new ErrorResponse(400, MemberConstants.INVALID_PERIOD), 400);
